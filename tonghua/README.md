# 同花 A股研究台

一个本地运行的 A 股研究工作台。新版入口参考 `daily_stock_analysis-main` 的工作流，改成“研究、组合、发现、回测、规则”五个工作区，并以决策仪表盘、策略能力包和真实数据约束为核心。

> 重要提示：本项目只用于个人研究和辅助决策，不构成投资建议。任何买卖决定都需要你结合自身风险承受能力独立判断。

## 主要功能

- 研究工作台：单股大图 K 线、核心结论、买点/止损/止盈、详细报告、数据明细和多角色复核集中展示。
- 决策仪表盘：参考 daily_stock_analysis 的报告结构，拆成核心结论、数据透视、情报、作战计划。
- 策略能力包：内置多头趋势、均线金叉、放量突破、缩量回踩、箱体震荡和风险排查。
- 组合扫描：批量扫描自选股，按操作、分数、趋势、动能和数据质量排序。
- 候选发现：优先使用当前可用的 AKShare 新浪实时全A快照，自动用真实单股历史行情补全可计算字段；真实数据不可用时停止，不使用演示候选池。
- 回测验证：用简单均线规则验证历史表现、收益、回撤、胜率和交易明细。
- 数据真实约束：行情、资金、财务、新闻、公告均来自真实接口；接口失败会提示或跳过。

## 运行方式

```powershell
pip install -r requirements.txt
powershell -ExecutionPolicy Bypass -File .\run_app.ps1
```

或直接运行：

```powershell
python -m streamlit run app.py
```

打开浏览器访问：

```text
http://localhost:8501
```

## 部署成网站

本项目已经整理成 Streamlit Cloud 可部署结构。把项目上传到 GitHub 后，在 Streamlit Cloud 里选择：

```text
入口文件：app.py
Python：3.12
依赖文件：requirements.txt
```

详细步骤见 [DEPLOY.md](DEPLOY.md)。

如果要给网站加访问密码，在 Streamlit Cloud 的 Secrets 中设置：

```toml
APP_PASSWORD = "你自己的密码"
```

如果要设置默认股票池，也可以在 Secrets 中加入：

```toml
APP_DEFAULT_CODES = "603906,600410"
```

页面左侧也有 `保存股票池` 按钮。保存后，把当前网址收藏起来，下次打开会自动带上这些股票。

## 推荐使用流程

1. 在左侧“股票池”输入关注的股票代码，比如 `600519, 000001, 300750`。
2. 进入“研究”，查看单股决策仪表盘、策略触发和关键价位。
3. 进入“组合”，批量扫描自选池，找出需要买入、观察、减仓或回避的标的。
4. 进入“发现”，从全市场快照筛出候选，再按需做深度复核。
5. 进入“回测”，验证当前规则在历史区间里的表现。
6. 如果已经持有，在左侧填写成本价和仓位，再看持仓建议。

## 速度说明

- 工作区只运行当前页面，不会在后台同时跑所有功能。
- 组合扫描使用轻量模式；研究页默认拉取完整详情，不再提供快速预览模式。
- 默认开启分钟线实时刷新，尽量把当天最新成交合入 K 线；如果接口不可用，页面会明确提示，不使用演示数据。
- “发现”会实时拉取全市场快照，并对展示候选自动补充可由历史行情真实计算的字段。
- 非手动刷新场景允许短缓存；点击“实时拉取市场快照并筛选”会绕过本地缓存。

## 参考项目

新版工作台重点参考了本地目录 `daily_stock_analysis-main`，上游项目为 `ZhuLinsen/daily_stock_analysis`，许可证为 MIT。当前项目没有直接依赖其完整 FastAPI/React 服务，而是吸收其工作台结构、报告 schema 思路和策略能力包设计，并用本项目已有真实数据接口落地。

## 优质股票发现是怎么筛的

这个功能借鉴 Qlib、FinRL 和 TradingAgents 这类开源项目常见的“先全市场打分，再对少数候选复核”的流程：

1. 直接使用当前可用的 AKShare 新浪实时全A快照；已确认不可用的全市场接口不再进入主流程，避免页面出现无意义的失败提示。
2. 先排除 ST、退市、新股标记、成交过低、极端估值、涨跌停附近和换手异常的股票。
3. 初筛分综合成交额、当日涨跌幅、历史补全后的60日涨跌幅、年初至今涨跌幅、换手率、振幅等真实字段；PE/PB、市值等无法从行情计算的字段不会补造。
4. 可切换 7 种筛选风格：稳健优质、趋势增强、低估修复、成长质量、资金关注、突破跟踪、超跌修复。
5. 对初筛前排再跑个股分析和多角色复核，最后给出“候选研究名单”，不是直接买卖指令。

## 每日报告与推送

手动生成报告：

```powershell
python scripts\daily_report.py --codes "600519,000001,300750"
```

报告会生成在 `reports` 目录。

如果要企业微信机器人推送：

```powershell
$env:WECHAT_WEBHOOK_URL="你的机器人Webhook"
python scripts\daily_report.py --codes "600519,000001,300750" --push
```

如果要邮件推送，需要设置：

```powershell
$env:SMTP_HOST="smtp.example.com"
$env:SMTP_PORT="465"
$env:SMTP_USER="your@email.com"
$env:SMTP_PASSWORD="邮箱授权码"
$env:REPORT_EMAIL_TO="to@email.com"
python scripts\daily_report.py --codes "600519,000001,300750" --push
```

可以用 Windows 任务计划程序每天收盘后运行上面的命令。
